# 입결 자동 수집 데이터 (대학어디가)

## 폴더 용도
대학어디가(adiga.kr) 자동 수집 프로그램이 생성한 학년도별 입결 Excel 파일을 보관합니다.

사용자 웹의 **입시결과 페이지** 가 이 데이터를 기반으로 표시됩니다.

## 파일 형식

### 파일명 규칙
```
adiga_입결_{입결 연도}.xlsx
```

**중요**: 파일명의 학년도는 **실제 입결이 발생한 연도** 입니다.
사용자 페이지에서는 자동으로 **그 다음 학년도 페이지** 에 "전년도 입시결과"로 표시됩니다.

예시:
- `adiga_입결_2026.xlsx` ← **2026학년도 입결** → 2027학년도 페이지에 표시
- `adiga_입결_2025.xlsx` ← 2025학년도 입결 → 2026학년도 페이지에 표시
- `adiga_입결_2024.xlsx` ← 2024학년도 입결 → 2025학년도 페이지에 표시

### Excel 구조 (37컬럼)
시트: `전년도입결`

| # | 컬럼 | 예시 |
|---|---|---|
| 1 | 대학명 | 서울대학교[본교] |
| 2 | 대학코드 | 0000019 |
| 3 | 전형유형 | 종합 / 교과 / 논술 / 수능 |
| 4 | 전형명 | 학생부종합전형(수시모집...) |
| 5 | 구분 | 수시 / 정시 |
| 6 | 모집단위 | 정치외교학부 |
| 7 | 모집인원 | 12 |
| 8 | 경쟁률 | 6.3 |
| 9 | 충원인원 | 1 |
| 10~15 | 학생부 환산점수/등급 50%, 70% | - |
| 16~37 | 백분위 50%, 70% (국어/수학/탐구/한국사/영어) | - |

## 운영 흐름

### 매년 갱신 절차 (관리자 페이지)
1. 대학어디가 자동 수집 프로그램 실행 → 새 Excel 생성
2. 관리자 페이지 → **입결 데이터 업로드** → 파일 선택 → 업로드
3. 백엔드가 자동으로:
   - 파일을 이 폴더에 영구 저장 (`adiga_입결_{year}.xlsx`)
   - DB 의 해당 학년도 데이터 삭제 후 새 데이터로 교체
4. 사용자 사이트에 즉시 반영

### 폴더의 역할 — 영구 보관 (백업·복원·재처리)
사용자가 직접 이 폴더에 파일을 둘 필요는 없습니다.
관리자 페이지에서 업로드된 파일이 이 폴더에 자동으로 저장됩니다.

폴더의 용도:
- **실수 복구**: 잘못된 Excel 업로드로 DB 손상 시 이전 파일로 복원
- **원본 보존**: 학년도별 원본을 보관해 추후 비교·검증 가능
- **재처리**: DB 손상·이전 시 폴더의 모든 Excel 을 한 번에 재 import 가능

### 재업로드 시 동작
같은 학년도 Excel 을 다시 올리면:
- 해당 학년도 (예: 2026) 기존 DB 데이터 모두 삭제
- 새 Excel 데이터로 완전 교체
- 다른 학년도 (예: 2027) 데이터는 영향 없음
