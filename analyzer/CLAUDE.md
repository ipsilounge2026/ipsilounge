# 학교생활기록부 분석 프로그램 (School Record Analyzer)

## 프로젝트 개요

학생부 PDF/이미지를 입력받아 분석하고, 내신 성적·세특·창체·행특·입결 비교를 종합한 **학생부 경쟁력 리포트**를 자동 생성하는 프로그램.

- **교육과정**: 2022 개정교육과정 (5등급제) 적용 학생 대상
- **실행 환경**: Claude Code (Max 플랜 토큰 사용, API 키 불필요)
- **입력**: 학생부 PDF + 이미지 혼용
- **출력**: Excel + PDF 리포트
- **지원 학과 옵션**: 학과 지정 / 미지정 모두 지원 (선택)

---

## 기술 스택

- Claude Code (Max 플랜, API 키 불필요) - 데이터 추출 + 분석 + 판단
- Python - 파싱 보조 + 데이터 정리 + 출력 포맷팅
- openpyxl - Excel 리포트 생성
- reportlab 또는 WeasyPrint - PDF 리포트 생성
- pdf2image - PDF → 이미지 변환
- kiwipiepy (0.23.1) - 한국어 형태소 분석 (키워드 추출용, G5 구현)
- wordcloud (1.9.6) + matplotlib (3.10.8) - 워드클라우드 시각화 및 이미지 저장 (G5 구현)

- `python generate_report.py <학생명>` 실행

## 학생부 구성 항목 및 대입 반영 여부

### 대입 반영 항목 (분석 대상)

| 항목 | 분석 영역 | 글자수 제한 |
|------|-----------|------------|
| 인적·학적 사항 | 기본 정보 추출 | - |
| 출결상황 | 출결 분석 | - |
| 창의적 체험활동 - 자율활동 | 창체 분석 | 연간 500자 |
| 창의적 체험활동 - 동아리활동 | 창체 분석 | 연간 500자 |
| 창의적 체험활동 - 진로활동 | 창체 분석 | 연간 700자 |
| 교과학습발달상황 (성적) | 내신 분석 | - |
| 교과학습발달상황 (세특) | 세특 분석 | 과목당 500자 |
| 행동특성 및 종합의견 | 행특 분석 | 연간 500자 |

### 대입 미반영 항목 (분석 제외)

수상경력, 독서활동상황, 진로희망분야, 영재·발명교육 실적, 자격증 및 인증 취득상황, 자율동아리, 개인봉사활동 실적은 모두 대입 미반영.

### 봉사활동 — [2026-04-17 G7 구현 완료]
- 점수화하지 않음
- 봉사시간만 별도 정리하여 리포트에 표기
- 학생 데이터 파일 `volunteer_data` 필드에 학년별 `{"hours": N, "activities": [..]}` 형식으로 저장
- Excel "출결·봉사" 시트 + PDF "출결 및 봉사활동" 섹션에 시수 + 주요 활동 표기
- 데이터 없으면 리포트에서 생략 (선택 필드)

### 출결 — [2026-04-17 G7 구현 완료]
- 학생부 기재 4종(결석/지각/조퇴/결과) × 3사유(질병/미인정/기타) 12칸으로 집계
- 입시 평가 관점: **미인정만** 성실성·규칙준수 역량 감점 요인. 질병/기타는 표기만.
- 감점 공식 (`config/config.yaml` `attendance_scoring`):
  - 미인정결석 -5/일, 미인정지각/조퇴/결과 -0.5/건
  - 기본 100점 - 감점 합계 = 최종 점수 (종합 경쟁력 공식의 출결 5% 에 반영)
- 데이터 없으면 만점 100점 가정 (종합 공식 영향 없음)

## 처리 파이프라인

### Step 0: 기존 리포트 확인 (사전 확인 단계) — [2026-04-17 G3+G4 구현 완료]

**분석 시작 전에 반드시 실행. output/ 폴더에서 동일 학생의 기존 리포트를 탐색.**

구현 지원:
- `modules/compare_generator.py`: 이전 Excel 파싱 + 판정 대상 자동 추출
- `generate_report.py`: `_v{N}` 파일명 자동 부여 (G4)
- `modules/qa_validator.py` P1-G / P2-G: compare_data 일관성 자동 검증
- `docs/checklist/analyzer/compare-data.yaml`: 판정 기준 (강화됨/유지됨/약화됨, 반영됨/부분반영/미반영)

#### 0-1. 기존 리포트 탐색
- `output/` 폴더에서 학생 이름이 포함된 파일(`{학생이름}_학생부분석_*.xlsx`, `{학생이름}_학생부분석_*.pdf`) 검색
- 동일 이름 파일이 없으면 → Step 1로 진행 (최초 분석)
- 동일 이름 파일이 있으면 → Step 0-2로 진행
- ※ 자동 탐색은 `modules.compare_generator.find_previous_reports()` 가 담당

#### 0-2. 동일 인물 확인 (Claude 대화형)
- 동일 학생 이름이라도 동명이인일 수 있으므로, 아래 정보를 대조하여 동일 인물 여부 확인:
  - **학교명**: 기존 리포트의 학교명(SCHOOL)과 현재 학생부의 학교명 비교
  - **생년월일/주민번호 앞자리**: 필요 시 대조 (학생 데이터 파일 미저장 시 Claude 가 학생부 원본에서 확인)
  - **학년 정보**: 기존 리포트의 학년과 현재 학생부의 학년이 연속되는지 확인
- 동일 인물이 아닌 경우 → Step 1로 진행 (최초 분석)
- 동일 인물이 확인된 경우 → Step 0-3으로 진행

#### 0-3. 이전 리포트 검토 (자동 파싱)
- `python -m modules.compare_generator <학생명>` 으로 이전 리포트 자동 파싱
- 추출 데이터:
  - 이전 영역별 등급 (세특/창체/행특 대표 등급)
  - **이전 핵심강점 3개** (종합요약 시트)
  - **이전 보완영역 3개** (종합요약 시트)
  - **역량별보완법 `fix_items` 전체** (역량별보완법 시트)
  - 성장 스토리 원문

#### 0-4. 이전 대비 변화 분석 (2회차 이상) — `compare_data` 필드에 기록

Claude 가 `docs/checklist/analyzer/compare-data.yaml` 체크리스트 기준으로 작성.
QA 가 enum/근거/글자수를 자동 검증하여 기준 흔들림 방지.

**필수 작성 대상:**

- **`grade_changes`** (자동 생성 가능, 수동 확인 권장): 영역별 등급 이전 vs 현재 (예: 세특 B→A ↑)
- **`strengths_tracking`**: **이전 핵심강점 3개 전부** tracking 필수
  - 각 항목에 `현재상태` enum: **강화됨 / 유지됨 / 약화됨**
  - `근거` 필드 필수 (학생부 원문 인용 또는 비교 분석)
- **`issues_tracking`**: **이전 보완영역 3개 + 역량별보완법 fix_items 전체** tracking 필수
  - 각 항목에 `상태` enum: **반영됨 / 부분반영 / 미반영**
  - `근거` 필드 필수
- **`new_strengths`** / **`new_issues`**: 이전 분석에 없었던 새 강점/보완점
- **`growth_comment`**: 최소 200자, 전반적 성장 방향 + 두드러진 변화 + 남은 과제 포함

**QA 자동 검증:**
| ID | 항목 | 실패 시 |
|---|---|---|
| P1-G-001 | strengths_tracking 개수 ≥ 이전 핵심강점 수 | 리포트 차단 |
| P1-G-002 | issues_tracking 개수 ≥ 이전 보완점 수(종합요약+fix_items) | 리포트 차단 |
| P1-G-003 | strengths_tracking[].현재상태 enum 검증 | 리포트 차단 |
| P1-G-004 | issues_tracking[].상태 enum 검증 | 리포트 차단 |
| P1-G-005 | 모든 tracking 항목 근거 필드 비어있지 않음 | 리포트 차단 |
| P2-G-001 | growth_comment 200자 이상 | WARN |
| P2-G-002 | new_strengths/new_issues 각 1개 이상 | WARN |

**리포트 반영 (자동):**
- Excel `이전분석대비변화` 시트 자동 생성 (compare_data 있을 때만)
- PDF `이전 분석 대비 변화` 섹션 (종합의견 직후 배치, compare_data 있을 때만)
- 파일명 `_v{N}` 접미사 자동 부여: `modules.compare_generator.get_next_version_number()` 가 `output/` 의 기존 파일 번호 + 1 자동 계산
  - 예: `연승훈_학생부분석_20260407_v2.xlsx`

### Step 1: 파일 읽기 & 전처리
- PDF → 페이지별 이미지 변환 (pdf2image)
- 이미지 파일 → 그대로 사용
- 결과: 페이지별 이미지 리스트

### Step 2: 데이터 추출 (Claude)
이미지를 Claude에게 넘겨서 구조화된 JSON으로 추출. 영역별 분리 요청:
1. 인적사항 + 출결
2. 교과 성적 (학기별) - 과목명, 학점, 원점수, 과목평균, 성취도, 성취도별분포비율, 석차등급, 수강자수
3. 세특 텍스트 (과목별)
4. 행특 텍스트 (학년별)
5. 창체 활동 (자율/동아리/진로)
6. 봉사활동 시수

### Step 3: 내신 성적 분석 (Python + Claude)

#### 3-1. 기본 성적 산출
- 학기별 평균 등급 산출 (학점 가중)
- 주요교과(국영수+계열교과) / 전교과 분리 분석
- 전형별 반영 내신 산출

#### 3-2. 원점수/평균/성취도별분포비율 맥락 분석
- 2022 개정교육과정 기준: 표준편차 없음, 성취도별 분포비율 활용
- 동일 등급이라도 원점수-평균 차이, 성취도별 분포에 따른 실질적 위치 분석
- 예: A등급이지만 A비율 5%인 과목 vs A비율 20%인 과목 구분

#### 3-3. 소인수 과목 표기
- 기준: 수강자 13명 이하 (config.yaml에서 조정 가능)
- 점수 보정은 하지 않고 리포트에 표기만

#### 3-4. 성적 추이 패턴 분석
5가지 유형 판별 (학년별 동일 비중, 가중치 없음):
- **상승형**: 학년이 올라갈수록 등급 향상 → 성장하는 모습으로 긍정 평가
- **V자 반등형**: 중간 하락 후 회복 → 위기 극복 스토리로 긍정 평가
- **역V자형**: 중간 상승 후 하락 → 후반 관리 부족으로 부정 평가
- **안정형**: 학년 간 등급 변동 미미 → 꾸준한 실력 유지
- **하락형**: 학년이 올라갈수록 등급 하락 → 가장 부정적

### Step 5: 세특 분석 (Claude)

  - **과목**: 과목명
  - **핵심 문장**: 학생부에 실제 작성되어 있는 문장을 그대로 발췌 (구체적 동기, 구체적 행동, 구체적 결과가 포함된 문장 우선 선정). 한 과목에서 여러 문장이 해당되면 모두 추출
  - **이유**: 해당 문장이 입시에서 높은 평가를 받을 수 있는 핵심 문장인 이유
  - **표현 역량**: 해당 문장이 학생의 어떤 역량(학업역량/진로역량/공동체역량의 세부 항목)을 잘 표현하고 있는지 서술

### Step 8.5: QA 검증 (Python, 자동) — 필수 단계

**분석 완료 후, 리포트 생성 전에 자동으로 실행. Step 9는 QA P1 전체 통과 시에만 진행.**

검증 모듈: `modules/qa_validator.py`

#### 검증 항목

**P1 — 필수 (FAIL 시 리포트 생성 차단)**
- 구조 완전성: 세특/창체/행특/연계성/핵심문장/보완법 데이터 존재 확인
- 점수 범위: 모든 개별 점수 1~10 범위 내
- 가중합산 수학 검증: 개별 점수 × 가중치 합계와 기록된 가중합산 일치 여부 (오차 0.05 이내)
- 등급 일치: 가중합산 점수 → 등급 변환이 등급 기준(S≥8.5, A≥7.0, B≥5.0, C≥3.5)과 일치

**P2 — 중요 (WARN, 품질 저하 위험)**
- 글자수 최소 기준: 세특 코멘트 200자↑, 창체 코멘트 300자↑, 행특 코멘트 200자↑, 연계성 200자↑, 보완법 300자↑
- 핵심문장 과목 커버리지: 모든 과목에 핵심문장(또는 "해당 없음") 존재, 4항목 완전성
- 교차 참조 일관성: setuek_data ↔ setuek_comments ↔ good_sentences 과목명 매칭
- 창체 분량 판정 정합성: 분량 판정(충실활용/적정활용/활용부족)이 비율 수치와 일치

**P3 — 참고 (INFO, 품질 개선 제안)**
- 코멘트 반복 검출: 여러 과목에서 동일 50자 이상 구절 반복 시 경고
- 등급-점수 정합성: 점수 분포와 등급 간 비정상적 괴리 검출

#### QA 결과 출력
- 콘솔에 PASS/FAIL/WARN 요약 출력
- FAIL 항목은 상세 내용 + 수정 방법 제시
- P1 전체 PASS → Step 9 자동 진행
- P1 FAIL 존재 → 리포트 생성 차단, 수정 필요 사항 안내

### 교과 이수 기준표 (data/course_requirements.xlsx)

**스키마**: 단일 시트 `권장과목`, 5개 열 (Wide + 쉼표 구분 형식).

| 대학 | 모집단위 | 핵심과목 | 권장과목 | 비고 |
|------|----------|----------|----------|------|
| 서울대 | 컴퓨터공학부 | 미적분, 기하, 물리학Ⅰ | 정보, 인공지능 기초, 데이터 과학 | 2025 기준 |
| 연세대 | 경제학부 | 미적분, 확률과 통계 | 경제, 사회문제 탐구 | |

**작성 규칙**:
- 한 모집단위 = 한 행 (대학·모집단위 단위로 한 줄)
- `핵심과목`/`권장과목` 셀에는 과목명을 **쉼표(,)** 로 구분해 나열
- 빈 셀이나 `-` 는 "해당 없음"
- 첫 행은 헤더, `※` 로 시작하는 행은 주석 처리되어 무시됨
- 대학별로 시점이 달라도 행 단위로 부분 추가/수정 가능

**공유 원칙**:
- 이 파일은 `school-record-analyzer/data/` 가 단일 원천(single source of truth)
- 입시라운지 백엔드(`ipsilounge/backend`)는 환경변수 `SHARED_DATA_ROOT` 또는
  자동 계산된 기본값을 통해 동일한 파일을 참조함
- 매년 대학 발표를 한 곳에서만 갱신하면 두 시스템 모두 반영됨

**평가 우선순위**:
1. 지원 대학+학과 지정 → 해당 대학+모집단위 행 필터
2. 학과만 지정 → 해당 모집단위 전체 대학 종합 (과반수 핵심 기준)
3. 미지정 → 이수 과목 패턴으로 적합 모집단위 역추정

### 평가 항목 및 가중치 (7개 항목, 총 100%)

| 번호 | 항목 | 가중치 |
|------|------|--------|
| ① | 교과연계성 | 17% |
| ② | 탐구동기 | 17% |
| ③ | 탐구과정 | 17% |
| ④ | 결과성찰 | 17% |
| ⑤ | 전공적합성 | 10% |
| ⑥ | 차별성 | 7% |
| ⑦ | 학업태도 | 15% |

### 채점 기준 (각 항목 1~10점)

**① 교과연계성 (17%)** — 해당 과목의 핵심 개념·단원에서 탐구가 시작되었는가
- 9~10점: 특정 단원/개념을 명시하고 거기서 심화 질문을 도출, 교과 핵심 역량과 탐구가 긴밀히 연결
- 7~8점: 특정 단원/개념에서 출발했으나 심화 질문 도출까지는 이르지 못함
- 5~6점: 과목과 관련은 있으나 어떤 개념에서 출발했는지 불명확
- 3~4점: 과목과의 연관이 느슨하고 교과 개념 연결이 피상적
- 1~2점: 과목과 무관하거나 단순 감상 수준

**② 탐구동기 (17%)** — 왜 이 주제에 관심을 가졌는지 구체적으로 드러나는가
- 9~10점: 수업 중 특정 상황/질문/현상에서 촉발된 동기가 매우 구체적이며 진정성이 드러남
- 7~8점: 구체적 동기가 있으나 촉발 상황의 서술이 다소 부족
- 5~6점: "관심을 가지게 되어" 수준의 일반적 동기
- 3~4점: 동기 언급이 형식적이거나 막연함
- 1~2점: 동기 언급 없이 바로 활동 나열

**③ 탐구과정 (17%)** — 어떤 방법으로, 어떤 자료를 활용해 탐구했는가
- 9~10점: 논문/실험/데이터 분석 등 구체적 방법론 + 단계적 과정 서술 + 변인 통제 등 체계적
- 7~8점: 방법론은 구체적이나 과정의 단계적 서술이 부분적
- 5~6점: "조사하였다", "분석하였다" 등 방법은 있으나 구체성 부족
- 3~4점: 과정 서술이 매우 간략하거나 단편적
- 1~2점: 과정 없이 결론만 제시하거나 단순 요약

**④ 결과성찰 (17%)** — 탐구를 통해 무엇을 알게 되었고, 어떤 변화가 있었는가
- 9~10점: 구체적 결론 + 한계 인식 + 후속 탐구 방향 + 다른 맥락으로의 확장 가능성까지 제시
- 7~8점: 구체적 결론 + 한계 인식 또는 후속 방향 중 하나는 제시
- 5~6점: 결론은 있으나 피상적 ("~의 중요성을 알게 됨"), 한계 인식이나 후속 방향 없음
- 3~4점: 결론이 모호하거나 탐구 내용과 직접 연결되지 않음
- 1~2점: 결론 없이 활동 나열로 끝남

**⑤ 전공적합성 (10%)** — 탐구 내용이 지원 전공과 연결되는가
- 9~10점: 전공 핵심 역량과 직결되는 탐구 + 학년 간 일관된 관심사 흐름 + 전공 심화 의지
- 7~8점: 전공 핵심 역량과 직결되나 일관성이 다소 부족
- 5~6점: 간접적 연관은 있으나 연결이 느슨
- 3~4점: 전공과의 연결이 피상적
- 1~2점: 전공과 무관
- ※ 지원 학과 미지정 시 이 항목 제외, 나머지 6개로 가중치 재배분

**⑥ 차별성 (7%)** — 다른 학생 대비 독창적이고 심화된 수준인가
- 9~10점: 독창적 주제 + 학술적 심화 + 교과서 이상의 내용 + 자기만의 관점/접근
- 7~8점: 독창적 요소가 있으나 학술적 심화는 부분적
- 5~6점: 성실하지만 보편적인 수준의 활동
- 3~4점: 일반적 수준이며 독창성이 거의 없음
- 1~2점: 교과서 내용 반복 또는 형식적 기술

**⑦ 학업태도 (15%)** — 수업 참여도, 질문/발표 적극성, 자기주도 학습 의지
- 9~10점: 적극적 수업 참여 + 자발적 질문/발표 + 자기주도적 심화 학습이 구체적 에피소드로 서술
- 7~8점: 적극적 참여와 자기주도 학습이 있으나 에피소드 구체성이 부분적
- 5~6점: 수업에 성실히 참여하나 적극성이 특별히 드러나지 않음
- 3~4점: 수업 참여는 언급되나 소극적이거나 형식적
- 1~2점: 수업 참여나 학습 태도에 대한 언급이 거의 없음

| 학업역량 - 탐구력 | 탐구동기, 탐구과정, 결과성찰 | 세특 + 창체(동아리, 자율, 진로) |
| 진로역량 - 교과이수 노력 | 전공 관련 과목 이수 현황 | 내신 분석 (교과이수 충실도) |
| 진로역량 - 교과성취도 | 전공 관련 과목 등급/성취도 | 내신 분석 |
| 진로역량 - 진로탐색 | 전공적합성, 진로탐색 구체성 | 세특 + 창체(진로, 동아리, 자율) |
| 공동체역량 - 협업·소통 | 협력, 의사소통 | 행특 + 창체(자율, 동아리) |
| 공동체역량 - 나눔·배려 | 타인 존중, 배려 태도 | 행특 |
| 공동체역량 - 성실성·규칙준수 | 출결, 규칙 준수 | 출결 + 행특 |
| 공동체역량 - 리더십 | 공동체 이끄는 역량 | 행특 + 창체(자율, 동아리) |

## 실행 흐름

1. `input/` 폴더에 학생부 파일 넣기
2. Claude Code에 분석 지시 (파일 경로 직접 지정 또는 input/ 폴더 자동 탐색)
3. **[자동] `output/` 폴더에서 동일 학생 기존 리포트 확인** (Step 0)
   - 기존 리포트 있으면 → 동일 인물 확인 → 이전 리포트 검토 후 분석 진행
   - 2회차 이상이면 분석 완료 후 "이전 분석 대비 변화" 섹션 자동 추가
4. (선택) 지원 대학·학과 지정 또는 미지정
5. (선택) 분석 범위 지정 — 전체 분석 / 내신 제외 / 특정 영역만 등
6. Claude Code가 파일 읽기 → 이미지 변환 → 데이터 추출 → 분석 수행
7. **[필수] `data/students/<학생명>.py` 파일에 분석 결과(15개 변수) 작성**
8. **[필수] `python generate_report.py <학생명>` 실행으로 Excel + PDF 리포트 생성** → `output/` 폴더에 저장
   - 분석만 하고 리포트를 생성하지 않는 것은 불완전한 실행임
   - 분석 결과는 반드시 Excel + PDF 두 가지 포맷으로 출력해야 함
   - 파일명 규칙: `{학생이름}_학생부분석_{날짜}.xlsx` / `{학생이름}_학생부분석_{날짜}.pdf`
   - 2회차 이상: `{학생이름}_학생부분석_{날짜}_v{횟수}.xlsx` (예: `연승훈_학생부분석_20260407_v2.xlsx`)

### 명령행 진입점 사용법


진입점 동작:
1. `data/students/<학생명>.py` 파일 동적 로드 (한글 파일명 지원)
2. 필수 변수 15개 검증 (누락 시 exit 2)
3. **Step 8.5: QA 검증** 자동 실행 (`modules/qa_validator.py`)
4. QA P1 통과 시 `create_excel(sd, xlsx_path)`, `create_pdf(sd, pdf_path)` 호출
5. 출력 경로는 진입점에서 자동 조립 (`output/{STUDENT}_학생부분석_{TODAY}.xlsx/.pdf`)

### 실행 모드별 처리

**CLI 옵션** (2026-04-19 구현):

```bash
python generate_report.py <학생명> [--mode <모드>] [--areas <영역목록>]
```

| --mode 값 | 분석 범위 | 리포트 포함 시트 |
|-----------|----------|----------------|
| `full` (기본값) | 내신+세특+창체+행특+입결 | 전체 시트 (내신/입결은 현재 미구현 → 실질 9~13개) |
| `no-grade` | 세특+창체+행특 (내신/입결 제외) | 내신분석·입결비교·전형적합도 시트 제외 (현재 NOOP, 내신 시트 구현 시 자동 스킵) |
| `partial` | `--areas` 로 지정된 영역만 | 선택 영역 시트 + 종합요약 + 교차영역 시트(연계성/대학평가요소/역량별보완법) |

**`--areas` 값** (쉼표 구분, `--mode partial` 시 필수):
- `setuek` → 세특분석·세특코멘트·핵심평가문장
- `changche` → 창체분석
- `haengtuk` → 행특분석

**사용 예시**:
```bash
python generate_report.py 연승훈                               # 기본 full
python generate_report.py 연승훈 --mode no-grade               # 내신 제외
python generate_report.py 연승훈 --mode partial --areas setuek # 세특만
python generate_report.py 연승훈 --mode partial --areas setuek,changche
```

**mode_config 동작**:
- `modules/mode_config.py` 의 `ModeConfig` 가 create_excel / create_pdf / run_full_qa 에 공통으로 전달
- partial 모드에서 미선택 영역은 **Excel 시트 생성 스킵** + **PDF 섹션 생성 스킵** + **QA 구조 검증을 INFO 로 스킵** (FAIL 로 차단하지 않음)
- 교차영역 시트(연계성/대학평가요소/역량별보완법) 는 partial 에서도 항상 포함
- 선택적 시트(키워드/출결·봉사/이전분석대비변화) 는 기존 데이터 유무 조건만 따름 (mode 와 독립)

### 부분 학년 학생부 지원

학생부가 일부 학년만 기록된 상태(예: 1학년 1학기만 있는 신규 상담)에서도 리포트 생성이 가능하다.

**지원 시나리오:**

| 시나리오 | 동작 |
|---------|------|
| 1학년 1학기만 (초기 상담) | 세특 몇 과목 + 자율활동만으로도 리포트 생성. 표지 "1학년" 자동 표시 |
| 1학년 2학기까지 | "1학년" 표시 |
| 1~2학년 (2학년 종료 시점) | "1~2학년" 표시 |
| 1~3학년 (전체) | "1~3학년" 표시 |

**자동 처리 항목:**
- **표지의 학년 범위** (`학교생활기록부 기록: ~학년`): `setuek_data`/`changche_data`/`haengtuk_data`에서 학년을 자동 감지하여 생성
- **세특 학년 순회**: 데이터에 존재하는 학년만 동적으로 순회 (하드코딩 없음)
- **비교과 종합 점수 계산**: 존재하는 데이터만으로 계산 (없는 학년은 자동 제외)

**유연한 QA 검증 규칙:**
- 세특: 최소 1과목 이상
- 창체: 최소 1행 이상 (자율/동아리/진로 중 일부만 있어도 OK)
- 행특: 최소 1학년 이상
- 연계성: 학년 1개면 최소 1항목, 2개 이상이면 2항목 권장
- 학년 1개뿐일 때 "학년별 성장 흐름" 항목은 생략 가능

**데이터 파일 작성 시 주의:**
- 학년 필드는 존재하는 학년만 입력 (1, 2, 3 중 일부)
- 학기 구분은 필요 없음. 1학기만 있어도 학년=1로 입력
- 창체는 존재하는 영역만 입력
- 연계성 데이터는 학년이 1개뿐이면 "세특 과목 간 연계", "창체-세특 연계" 2항목으로 충분

### 리포트 생성 필수 원칙
- 분석이 완료되면 **사용자 추가 지시 없이도** 자동으로 리포트 생성 단계로 진행
- 분석 결과를 대화창에 텍스트로만 출력하고 끝내는 것은 금지
- 리포트 생성 중 오류 발생 시, 오류를 해결한 뒤 반드시 재시도하여 리포트 출력 완료
- 리포트 생성 완료 후 파일 경로를 사용자에게 안내
