## 3. 페이지 구조 설계

### 3-1. 사용자 유형별 접근 구조

| 페이지 | 용도 | 접근 권한 |
|--------|------|-----------|
| 강사 브랜드 사이트 | 강사 홍보, 학생 모집 | 누구나 (비로그인) |
| 학생/학부모 마이페이지 | 모든 과목 성적/과제/클리닉 통합 확인 | 로그인 (학생/학부모) |
| 강사 관리 페이지 | 본인 학생 성적/출결/채점 관리 | 로그인 (강사) |
| 관리자 페이지 | 전체 강사/학생/서비스 관리 | 로그인 (관리자) |

### 3-2. 강사 브랜드 사이트 (공개)

- URL: `강사이름.ipsilounge.co.kr`
- 서브도메인 기반, 강사별 독립 페이지
- 누구나 접근 가능 (학생 모집 목적)

```
kimjieun.ipsilounge.co.kr (공개 페이지)
├── 강사 소개, 경력
├── 수업 안내, 시간표
├── 수강 후기
└── 수강 신청 / 문의
```

구현 요소:
- 강사가 관리 페이지에서 로고, 색상, 소개글 설정
- 자동으로 서브도메인 페이지 생성
- Vercel 와일드카드 도메인 설정 필요

### 3-3. 학생/학부모 마이페이지 (로그인 필수)

- URL: `ipsilounge.co.kr/mypage`
- 수강 중인 모든 과목의 정보를 한 페이지에서 확인

```
내 페이지
├── 수학 (김지은 선생님) → 성적, 과제, 클리닉
├── 영어 (박수진 선생님) → 성적, 과제, 클리닉
├── 국어 (이현우 선생님) → 성적, 과제, 클리닉
├── 전체 출결 현황
├── 학생부 분석 리포트 (기존 기능)
└── 상담 예약 (기존 기능)
```

핵심: 학생이 3과목을 듣더라도 각 강사 페이지에 따로 들어갈 필요 없이 한 화면에서 전부 확인.

### 3-4. 강사 관리 페이지 (강사 로그인)

- URL: `kimjieun.ipsilounge.co.kr/manage` 또는 `ipsilounge.co.kr/teacher`

```
강사 관리 페이지
├── 내 학생 목록
├── 시험 등록 / OMR 채점
├── 성적 통계
├── 클리닉 예약 관리
├── 과제 관리
├── 출결 관리
└── 내 브랜드 사이트 설정
```

## 4. 추가 기능 상세

### 4-1. OMR 자동 채점 (오프라인 시험)

핵심 기능. 오프라인 시험지를 스캔/촬영하여 자동 채점.

**작동 플로우:**
1. 강사가 시험 등록 (시험명, 문항수, 정답 입력)
2. OMR 답안지 출력 (우리가 제공하는 표준 양식)
3. 학생이 시험 보고 OMR에 마킹
4. 강사/조교가 스캔 또는 앱으로 촬영해서 업로드
5. 자동 채점 → 통계 생성 → 학생/학부모에게 배포

**구현 방식 옵션:**

| 방식 | 정확도 | 비용 | 난이도 | 비고 |
|------|--------|------|--------|------|
| 전통 OMR (OpenCV) | 99%+ | 거의 없음 | ⭐⭐⭐ | 정해진 양식 필요, 가장 안정적 |
| Claude Vision | 90~98% | 월 수만~십수만원 | ⭐⭐ | 양식 자유도 높음, 검수 필요 |
| 온라인 시험 (학생 직접 입력) | 100% | 없음 | ⭐ | OMR 불필요, 보조 옵션으로 병행 |

**추천: 전통 OMR (OpenCV)** - 정해진 답안지 양식 제공 + 이미지 처리로 자동 인식
- 앱에서 카메라 촬영 → 바로 업로드도 가능
- 온라인 시험은 보조 옵션으로 병행 제공

**채점 후 자동 생성 통계:**
- 반 평균, 등수, 등급컷
- 문항별 정답률
- 선택지별 비율
- 학생별 성적 추이 (시험 간 비교)

### 4-2. 성적 관리

- 시험별 점수 기록 (자동 채점 또는 수동 입력)
- 학생별 성적 추이 그래프
- 반 평균, 등수, 등급 자동 계산
- 학생/학부모에게 자동 배포

### 4-3. 클리닉(보충수업) 예약

- 강사가 클리닉 가능 시간대 등록
- 학생이 앱/웹에서 셀프 예약/변경
- 성적 결과와 연동 (시험 후 자동 클리닉 안내)
- 현재 상담 예약 시스템 구조 활용 가능

### 4-4. 출결 관리

- 날짜별 출석/결석/지각 체크
- 학부모 자동 알림 (결석 시 즉시 알림)
- 월별 출결 통계

### 4-5. 과제 관리

- 강사가 과제 출제 (텍스트 + 파일 첨부)
- 학생이 앱/웹에서 제출 (파일/텍스트)
- 제출 현황 추적 (미제출자 알림)
- 과제 점수/피드백 기록

### 4-6. 통합 알림

- 성적 등록 시 자동 알림
- 출결 알림 (결석 시 학부모에게)
- 과제 마감 알림
- 클리닉 예약 확인/변경 알림
- 공지사항 알림
- 앱 푸시 + 웹 알림

### 4-7. 영상 관리 (추후)

- 강의 영상 업로드 + 수강 추적
- 학생별 재생 진도 확인
- 영상 보안 (워터마크/서명 URL) - 수익 발생 후 추가

## 7. 기술 고려사항

### 서버 사양
- 현재: AWS EC2 t3.micro (1 vCPU, 1GB RAM)
- OMR 이미지 처리, 영상 관리 추가 시 사양 업그레이드 필요
- 최소 t3.small (2GB RAM) 이상 권장

### 스토리지
- 현재: EC2 로컬 (SQLite)
- 확장 시: AWS S3 (이미지/영상 저장) + RDS (DB 분리) 검토
- OMR 이미지만이면 S3 비용 미미 (이미지 1장 수백KB)
- 영상 추가 시 S3 비용 증가 (영상 1개 500MB~2GB)

### 앱 (Flutter)
- 모든 기능을 앱에서도 사용 가능하도록 설계
- OMR 촬영: 앱 카메라로 답안지 촬영 → 업로드
- 푸시 알림: Firebase Cloud Messaging (FCM)
