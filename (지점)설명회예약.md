## 1. 시스템 개요

학원 각 지점에서 설명회 일정을 예약 신청하고, 관리자가 이를 승인·관리하는 웹 기반 예약 시스템입니다.

- **접속 방법**: 배포된 Google Apps Script 웹앱 URL 접속
- **사용 대상**: 학원 각 지점 담당자(사용자), 본원 관리자
- **데이터 저장소**: Google Sheets (DB 역할)

## 3. 데이터 구조

| 컬럼 | 설명 |
|------|------|

## 4. 사용자 기능

### 4-1. 회원가입

- 입력 항목: 지점명, 이메일, 비밀번호 (8자 이상)
- 이메일 중복 확인 후 등록
- 비밀번호는 SHA-256으로 암호화 저장
- 가입 후 로그인 필요 (자동 로그인 없음)

### 4-2. 로그인 / 로그아웃

- 이메일 + 비밀번호로 로그인
- 세션 유효 시간: 24시간
- 로그아웃 시 세션 즉시 삭제, 입력 필드 초기화
- 로그인 시 아이디 저장, 로그인 유지 선택
- 브라우저 탭 닫아도 세션 유지 (sessionStorage 기반)

### 4-3. 비밀번호 찾기

```
이메일 입력
→ 6자리 인증번호 이메일 발송 (10분 유효)
→ 인증번호 확인
→ 새 비밀번호 설정
```

### 4-4. 캘린더 - 설명회 일정 확인

- 월별 캘린더 뷰 제공
- **신청 가능한 날짜에만** 설명회 배지 표시
  - 오전/오후/저녁 모든 시간대 마감 시 해당 날짜 미표시
  - 신청 불가 날짜, 마감일 지난 날짜 미표시
  - 비공개 설정된 일정 미표시
- 여러 설명회가 있을 경우 색상으로 구분

### 4-5. 예약 신청

```
캘린더에서 설명회 날짜 클릭
→ 시간대 선택 (오전/오후/저녁, 잔여 수 표시)
→ 담당자 이름, 연락처, 참석 예정 인원 입력
→ 요청사항 입력 (선택)
→ 예약 신청 완료 → status: pending (대기)
```

- 동일 날짜+시간대 중복 신청 방지
- 시간대별 정원 초과 시 신청 불가
- Lock 처리로 동시 신청 충돌 방지

### 4-6. 예약 수정

- 대기(pending) / 수정대기(modified) / 승인(approved) 상태 모두 수정 가능
- 수정 가능 항목: 날짜, 시간대, 담당자 이름, 연락처, 참석 예정 인원, 요청사항
- **수정 사유 입력 필수**
- 수정 완료 시 status → `modified` (수정대기) 로 변경
- 관리자에게 재승인 요청 알림 메일 자동 발송

### 4-7. 예약 취소

- 대기/수정대기/승인 상태 모두 취소 가능
- **취소 사유 입력 필수**
- 취소 완료 시 status → `cancelled`
- 신청자에게 취소 안내 메일 자동 발송

### 4-8. 내 예약 현황

- 전체 예약 내역 조회 (최신순)
- 상태별 색상 구분
  - 대기: 노란색
  - 수정대기: 보라색
  - 승인: 초록색
  - 취소: 회색 (흐리게)
- 수정/취소 사유 표시

## 5. 관리자 기능

### 5-1. 대시보드

- **통계 카드**: 전체 예약 / 승인 대기 / 수정 대기 / 승인 완료 / 취소 / 총 신청인원 / 총 실제참석
- **지점별 참석 현황**: 지점명, 예약 수, 신청 인원, 실제 참석 인원
- **일정별 현황**: 설명회명, 기간, 승인 수, 신청 인원, 실제 참석 인원

### 5-2. 예약 관리

- 전체 예약 신청 목록 조회
- 필터: 상태별(전체/대기/수정대기/승인/취소), 지점명 검색
- **예약 승인**: 대기(pending) / 수정대기(modified) 상태 → 승인 처리 → 신청자에게 확정 메일 발송
- **예약 취소**: 모든 상태 취소 가능 → 신청자에게 취소 메일 발송
- **실제 참석 인원 입력**: 승인된 예약에 한해 직접 입력 (데이터 누적 목적)
- **수정/취소 사유 확인**: 클릭 시 전체 내용 토글 표시

### 5-3. 일정 관리

#### 일정 등록/수정

- 입력 항목
  - 설명회 제목, 설명 내용
  - 신청 가능 기간 (시작일 ~ 종료일)
  - 신청 불가 날짜 (기간 내 캘린더에서 클릭으로 선택/해제)
  - 시간대별 최대 예약 수 (오전/오후/저녁, 0 = 미운영)
  - 예약 마감일시
  - 공개/비공개 설정

#### 공개/비공개 토글

- 공개: 사용자 캘린더에 노출
- 비공개: 사용자 화면에서 숨김 (관리자만 확인 가능)

#### 일정 삭제

- 삭제 후 복구 불가

### 5-4. 메일 발송

#### 발송 탭

- **발송 대상 설명회 선택** (체크박스, 복수 선택 / 미선택 시 전체)
- **발송 대상 지점 선택** (체크박스, 복수 선택 / 미선택 시 전체)
  - 지점 목록: 승인된 예약자 중 자동 추출
- **발송 대상 미리보기**: 선택 조건에 따른 발송 예정 인원 실시간 표시
- 메일 제목 / 내용 입력
- 발송 완료 후 자동으로 이력 탭 전환

#### 이력 탭

- 발송 이력 카드 목록 (최신순)
  - 메일 제목, 발송일시, 설명회, 지점, 내용 미리보기, 발송 수
- 카드 클릭 → 상세 팝업
  - 발송 조건 전체 정보
  - 메일 내용 전체
  - 수신자 전체 목록 (지점명 + 이메일)

## 6. 예약 상태 흐름

```
신규 신청
    ↓
pending (승인 대기)
    ↓           ↘
관리자 승인    사용자/관리자 취소
    ↓               ↓
approved         cancelled
(승인 완료)      (취소)
    ↓
사용자 수정 요청
    ↓
modified (수정 대기)
    ↓           ↘
관리자 재승인  관리자/사용자 취소
    ↓               ↓
approved         cancelled
```

### 상태별 가능한 액션

| 상태 | 사용자 수정 | 사용자 취소 | 관리자 승인 | 관리자 취소 |
|------|:---------:|:---------:|:---------:|:---------:|
| pending (대기) | ✅ | ✅ | ✅ | ✅ |
| modified (수정대기) | ✅ | ✅ | ✅ | ✅ |
| approved (승인) | ✅ | ✅ | - | ✅ |
| cancelled (취소) | ❌ | ❌ | ❌ | ❌ |

## 7. 메일 발송 로직

| 발송 시점 | 수신자 | 내용 |
|-----------|--------|------|
| 관리자 예약 승인 시 | 신청 지점 | 예약 확정 안내 (설명회명, 날짜/시간대, 담당자, 인원) |
| 사용자/관리자 취소 시 | 신청 지점 | 예약 취소 안내 (취소 사유 포함) |
| 사용자 수정 시 | 관리자 | 재승인 요청 알림 (기존/변경 날짜, 수정 사유) |
| 비밀번호 찾기 시 | 본인 | 6자리 인증번호 (10분 유효) |
| 관리자 공지 발송 시 | 선택된 지점 | 자유 형식 공지 |

> **메일 실패 처리**: 모든 메일 발송은 try-catch로 감싸져 있어 메일 발송 실패 시에도 예약 처리(승인/취소/수정)는 정상 완료됩니다.

## 8. 보안 및 최적화

### 보안

- **비밀번호 암호화**: SHA-256 해시 저장 (복호화 불가)
- **세션 인증**: 모든 API 요청 시 sessionId 검증
- **권한 분리**: 관리자 API는 role === 'admin' 검증
- **세션 만료**: 24시간 후 자동 만료

### 동시성 제어

- **LockService**: 예약 생성/수정 시 스크립트 락 획득 (10초 대기)
- 동시 신청으로 인한 정원 초과 방지

### 성능 최적화

- **CacheService**: 공개 일정 조회 결과 5분 캐시
- 예약 생성/수정/취소 시 캐시 즉시 초기화
- 신청 가능 날짜 서버에서 사전 계산 후 전달

## 9. 초기 설정 가이드

### 9-1. Google Sheets 준비

1. 새 Google Sheets 생성
2. URL에서 `/d/` 뒤 ID 복사
   ```
   https://docs.google.com/spreadsheets/d/[여기가 ID]/edit
   ```

### 9-2. GAS 프로젝트 설정

1. Sheets → 확장 프로그램 → Apps Script
2. `Code.gs` 상단 설정값 교체
   ```javascript
   const SHEET_ID = '복사한 스프레드시트 ID';
   const ADMIN_EMAIL = '관리자 이메일 주소';
   ```
3. `index.html` 하단 GAS_URL 교체
   ```javascript
   const GAS_URL = '배포 후 웹앱 URL';
   ```

### 9-3. 관리자 계정 생성 (최초 1회)

GAS 에디터에서 `createAdminAccount` 함수 실행
- 초기 비밀번호: `admin1234!`
- 로그인 후 비밀번호 찾기로 변경 권장

### 9-4. 배포

1. 배포 → 새 배포 → 유형: 웹 앱
2. 실행 사용자: **나**
3. 액세스 권한: **모든 사용자**
4. 배포 URL 복사 → `index.html` GAS_URL에 입력
5. 다시 저장 → 배포 관리 → 새 버전으로 재배포

### 9-5. 코드 수정 후 반영 방법

```
코드 수정 → 저장
→ 배포 → 배포 관리 → ✏️ 수정
→ 버전: 새 버전 선택
→ 배포 클릭
```

> ⚠️ 단순 저장만으로는 반영되지 않습니다. 반드시 새 버전으로 재배포해야 합니다.
